Plugin Example


This plugin is a simple example of plugin. It aims to help you understand:
  • how to create custom components
  • how to use plugins in a SOFA application
  • how to create your own plugin

Components

MyFakeComponent is one of the simple possible components. It contains two attributes and does basically nothing (Tode: make it do something). The class appears at several places in the plugin:

  • as usual .h and .cpp files
  • these are registered in the project file (.pro) which can be opened using QtCreator
  • it is registered in the plugin, in file initPluginExample.cpp

OtherFakeComponent it templated on the type of degrees of fredom it processes. Note that the generic implementations are defined in a .inl file, registered as a header in the .pro file. The .cpp contains the instantiations of the class on different types, which generate binary code.

PendulumMapping is a real example of mapping. It shows examples of how to access the state vectors in the member methods.

CustomDataUnsignedWidget shows how to create a custom data widget in the Qt GUI.

Use the plugin in a SOFA application

To use this plugin, first compile it, then in the SOFA gui use Edit/Plugin Manager/Add. You need to do this only once.

Create your own plugin

The easiest way it to copy and rename the plugin example directory, then to suppress or rename classes.
To rename a plugin, you can use the changeClassName script (see below) to change the name of the plugin the same way as you change the name of a class.
To remove a class, delete its code files, and remove it from the .pro and initPluginExample.cpp files.

Rename a class

The Unix shell script changeClassName.sh allows you to easily change a class Name:
changeClassName.sh OldClassName NewClassName
replaces string OldClassName with string NewClassName in all files (including .pro) and file names in the current directory
